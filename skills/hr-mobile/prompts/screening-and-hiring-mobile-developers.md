# Screening and hiring mobile developers

Use these prompts when recruiting, screening, or interviewing mobile engineering candidates.

- "Create a phone screen question set for a [React Native/Flutter/Native iOS/Native Android] developer that I can use as a non-technical recruiter to filter candidates effectively."
- "Write a job description for a [Senior React Native Engineer/Staff iOS Developer/Mid-level Android Developer] that attracts strong candidates without setting unrealistic expectations."
- "What should I look for when reviewing a mobile developer's resume — what signals indicate real production experience versus tutorial projects?"
- "Explain what [Jetpack Compose/SwiftUI/Flutter Widgets/React Native New Architecture] means in plain language so I can ask informed follow-up questions in a screen call."
- "Create a hiring scorecard for a [Senior Mobile Engineer] role covering technical depth, platform knowledge, deployment experience, and collaboration skills."
- "What are the most common red flags in a mobile developer's resume or GitHub profile that indicate they overstate their experience?"
- "Design a take-home technical assessment for a [React Native/Flutter] role that tests real production skills — not algorithm puzzles — and can be completed in under 3 hours."
- "How do I distinguish between a Mobile Developer who is strong on one platform versus one who is genuinely cross-platform capable?"
