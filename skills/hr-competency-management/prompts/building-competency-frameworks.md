# Building competency frameworks

Use these prompts when designing or overhauling a competency framework for an organization or a specific function.

- "Design a core competency framework for a [technology company/professional services firm/manufacturing organization] with 5-7 enterprise-wide competencies that apply across all roles."
- "Define behavioral indicators at four proficiency levels — developing, proficient, advanced, expert — for the competency [stakeholder influence/analytical thinking/change leadership]."
- "Build a functional competency library for the [Finance/Sales/Engineering] function covering the 8-10 competencies that differentiate high performers in that domain."
- "Create a role-specific competency profile for a [Senior Product Manager/Regional HR Business Partner/Customer Success Manager] including required proficiency levels."
- "How should we differentiate between core competencies that apply to all employees, leadership competencies that apply to managers, and functional competencies that apply to specific roles?"
- "Audit our existing competency framework for [redundancy/missing leadership behaviors/overlap with values] and recommend a streamlined structure."
- "Design a competency validation process using manager interviews, top performer analysis, and job analysis data before we finalize the framework."
