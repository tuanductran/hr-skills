# Scenario: Hiring a Senior Site Reliability Engineer

## Context

A scaling e-commerce company needs to hire its first Senior SRE to own uptime and incident response as traffic grows, and the recruiter has limited SRE hiring experience.

## Step 1: Write the job description

Sample prompt: "Write a job description for a Senior Site Reliability Engineer role with clear on-call expectations."

Expected response: A job description emphasizing ownership of uptime/SLOs, incident response leadership, and infrastructure automation, with transparent on-call rotation details to set expectations upfront.

## Step 2: Build the interview loop

Sample prompt: "Draft a technical interview loop structure for a Cloud Engineer role, including panel focus areas" adapted for SRE, and "Design a live system-design exercise appropriate for a senior cloud architecture interview."

Expected response: A loop with a systems-design round (designing for high availability), a hands-on troubleshooting exercise, and a behavioral round focused on incident leadership.

## Step 3: Assess and reference-check

Sample prompt: "Create an interview scorecard rubric covering infrastructure-as-code, scalability, and security topics" and "Draft reference-check questions focused on a candidate's production incident response experience."

Expected response: A scorecard rubric with clear criteria per round, plus reference questions asking specifically about the candidate's role during a significant past outage and what they changed afterward.

## Workflow summary

The recruiter and hiring manager build a transparent job description, a structured multi-round interview loop calibrated for senior scope, and reference checks that validate real incident-response leadership experience.
