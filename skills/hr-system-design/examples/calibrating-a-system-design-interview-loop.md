# Calibrating a System Design Interview Loop

## Context

An engineering org has grown quickly and now has interviewers with wildly different bars for system design rounds, leading to inconsistent hiring decisions.

## Step 1: Build the rubric

Sample prompt: "Build an evaluation rubric for a system design interview covering scalability, trade-offs, and communication."

Expected response: A rubric with explicit scoring dimensions (requirements gathering, scalability/reliability trade-offs, data modeling, communication) each scored on a defined scale, replacing a single holistic gut score.

## Step 2: Create calibration materials

Sample prompt: "Draft example 'strong', 'adequate', and 'weak' answer patterns for a given system design prompt."

Expected response: Sample answer patterns illustrating what a strong vs. weak response looks like at each rubric dimension, used as a shared reference during calibration sessions.

## Step 3: Run calibration sessions

Sample prompt: "Create a calibration session agenda to align interviewers on scoring standards."

Expected response: An agenda where interviewers independently score the same sample transcript, then discuss scoring differences against the rubric until they converge on a shared bar.

## Workflow summary

The org builds a multi-dimensional rubric, creates calibration reference materials, and runs joint scoring sessions so interviewers converge on a consistent bar before continuing to run live loops.
