# Screening and hiring QA engineers

Use these prompts when recruiting, screening, or interviewing Quality Assurance and Quality Engineering candidates.

- "Create a phone screen question set for a [QA Automation Engineer/Manual QA Tester/SDET] that I can use as a non-technical recruiter to effectively filter candidates."
- "Write a job description for a [Senior QA Automation Engineer/Lead QA Engineer/Quality Engineer] that clearly distinguishes automation from manual testing expectations."
- "What should I look for when reviewing a QA engineer's resume — what signals distinguish a strong automation engineer from someone who only has manual testing experience?"
- "Explain what [Playwright/Selenium/Cypress/Appium] is in plain language so I can ask informed follow-up questions on a screen call."
- "Create a hiring scorecard for a [Senior QA Automation Engineer] covering automation depth, CI/CD integration experience, testing strategy, and quality culture contribution."
- "What are the most common mismatches between QA job descriptions and what companies actually need — and how do I write a more honest JD?"
- "Design a portfolio review checklist for a QA candidate's GitHub profile that a non-technical HR manager can use to assess quality of work."
- "How do I distinguish between [Manual QA, Automation Engineer, SDET, and Quality Engineer] when a hiring manager uses these terms interchangeably?"
