# Hiring a Smart Contract Security Engineer

## Context

A DeFi protocol team needs to hire a security engineer to review smart contracts before mainnet deployment, and wants to avoid hiring based on hype alone.

## Step 1: Write the job description

Sample prompt: "Draft a job description for a blockchain security engineer role focused on smart contract audits."

Expected response: A job description specifying required experience with EVM-based contract auditing, familiarity with common vulnerability classes, and prior audit or bug-bounty track record as a strong signal.

## Step 2: Design the technical exercise

Sample prompt: "Design a code-review exercise asking a candidate to identify vulnerabilities in a sample smart contract."

Expected response: A take-home or live exercise presenting a deliberately flawed contract (e.g., containing a reentrancy bug) and asking the candidate to identify and explain the vulnerability and propose a fix.

## Step 3: Verify experience

Sample prompt: "Draft reference-check questions focused on a candidate's shipped, audited smart contracts or protocols."

Expected response: Reference questions asking a past colleague or engagement lead to describe specific vulnerabilities the candidate caught (or missed) during a real audit engagement.

## Workflow summary

The team writes a specific, non-hype job description, tests real security review skill through a hands-on exercise, and verifies claimed experience through targeted reference checks focused on real audit outcomes.
