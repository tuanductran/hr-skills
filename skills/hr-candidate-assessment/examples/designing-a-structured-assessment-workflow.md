# Designing a structured assessment workflow

## Context

A hiring team needs a standardized technical work sample assessment for Senior Backend Engineer candidates to replace unstructured interview questions and reduce evaluation bias.

## Step 1: Define assessment objectives and competencies

Sample prompt: "Draft a work sample assessment brief for a Senior Backend Engineer role evaluating API design, concurrency, and error handling."

Expected response: A concise 2-hour assessment brief outlining a real-world API development scenario with explicit requirements and acceptable constraints.

## Step 2: Build the evaluation rubric

Sample prompt: "Create a 4-level scoring rubric evaluating code structure, test coverage, and documentation for a technical assessment."

Expected response: A rubric table with clear behavioral anchors for Needs Improvement, Meets Standards, Exceeds Standards, and Exceptional ratings across four core technical categories.

## Step 3: Candidate communication and debrief

Sample prompt: "Draft candidate invitation instructions and interviewer debrief guidelines for a technical assessment stage."

Expected response: Clear candidate setup instructions setting expectations on time commitment, submission format, and review timelines, plus a structured debrief template for the interview panel.

## Workflow summary

The talent team designs a realistic take-home work sample test, establishes objective scoring criteria, and provides candidates with clear instructions to ensure fair, consistent candidate evaluation.
